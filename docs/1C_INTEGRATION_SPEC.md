# Технічне завдання: Інтеграція 1С/BAF -> OneBox Hub (v1.0)

## Мета
Передача даних про роздрібні продажі (чеки) з 1С (УНФ 1.6) у Хаб інтеграцій для подальшого створення угод у OneBox CRM.

---

## 1. Налаштування на стороні 1С
В модулі Робочого Місця Касира (РМК) або в обробці проведення Чека ККМ необхідно додати процедуру відправки HTTP-запиту після успішного пробиття чека.

### Параметри підключення:
- **Метод:** `POST`
- **URL:** `https://[HUB_IP_OR_DOMAIN]/api/v1/webhook/1c`
- **Content-Type:** `application/json`
- **Авторизація:** `X-API-Key: [SECRET_KEY_FROM_HUB]` (буде надано після розгортання)

---

## 2. Структура JSON-пакета (Контракт)

1С має сформувати та відправити об'єкт наступної структури:

```json
{
  "receipt_id": "ЧЕК-000123",        // Унікальний номер чека в 1С
  "uuid": "550e8400-e29b-41d4...",   // Внутрішній UUID документа (для запобігання дублям)
  "date_time": "2026-03-10T16:45:00", // Дата та час пробиття
  "store": {
    "id": "STORE_001",               // ID магазину/складу
    "name": "Corner RiverMall"        // Назва точки продажу
  },
  "customer": {
    "phone": "380971657681",         // ТЕЛЕФОН (Обов'язково для матчингу)
    "last_name": "Жлуктинська",      // ПРІЗВИЩЕ (Для оновлення/створення контакту)
    "first_name": "Альона"           // Ім'я (опційно)
  },
  "payment": {
    "sum": 2990.00,                  // Загальна сума чека
    "method": "card",                // Спосіб оплати (card/cash)
    "currency": "UAH"
  },
  "items": [                         // Склад чека (масив товарів)
    {
      "product_id": "33426",         // ID товару в 1С
      "sku": "30666915-50ml",        // Артикул
      "name": "Парфуми Bombshell St. Tropez", // Назва товару
      "quantity": 1,                 // Кількість
      "price": 2990.00,              // Ціна за одиницю
      "discount": 0                  // Сума знижки на позицію
    }
  ]
}
```

---

## 3. Логіка обробки в Хабі
1. **Матчинг клієнта:** Хаб шукає в OneBox контакт за номером телефону. Якщо знайдено — угода прив'язується до нього. Якщо ні — створюється новий контакт (Телефон + Прізвище).
2. **Створення угоди:** В OneBox створюється нова угода (Order) у відповідному бізнес-процесі ("Продаж").
3. **Наповнення товарами:** Товари з масиву `items` додаються в угоду. Якщо товар за артикулом (`sku`) знайдено в OneBox — він прив'язується до картки продукту.
4. **Статус:** Угода створюється в статусі "Виконано" (або іншому узгодженому статусів для чеків).
