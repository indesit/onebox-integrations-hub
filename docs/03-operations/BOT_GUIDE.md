# Telegram Bot — Керівництво оператора

Last updated: 2026-03-23

## Огляд

Telegram-бот для власника та команди магазинів Secret Shop. Дає доступ до:
- касових залишків (1С)
- зведення продажів по магазинах
- картки покупця з повною історією
- статусу черги синхронізації OneBox

Технологія: **aiogram 3.x**, long-polling, FSM для форм вводу.

---

## Запуск

Бот запускається автоматично як сервіс `bot` в docker-compose:

```bash
docker compose up -d bot
docker compose logs -f bot
```

Перезапуск після змін коду (volume `./src` примонтований — код підхоплюється):

```bash
docker compose restart bot
```

## Авторизація

1. Користувач надсилає `/start`
2. Бот просить надіслати контакт (кнопка "📱 Надіслати номер телефону")
3. Телефон нормалізується до формату `+380XXXXXXXXX`
4. Порівнюється з `config/roles.yml`
5. Роль зберігається в `data/bot_users.json` (persistent)

**Live-reload ролей:** зміни в `config/roles.yml` діють одразу, без рестарту бота.

## Команди детально

### 💰 Залишок грошей в касах (`/cash`)
Запитує баланс кас в реальному часі з 1С через API.
Тільки для owner.

### 📈 Продажі (`/sales`)
Показує inline-кнопки → **Вчора** / **Сьогодні**.
Результат: виторг по кожному магазину за обраний день.
Джерело: `fact_sales_receipt_items` JOIN `dim_stores`.

### 👤 Покупець (`/customer` або кнопка)
Запитує номер телефону через ForceReply (Telegram відкриває поле вводу).
Повна картка клієнта:
```
🔍 Клієнт 380XXXXXXXXX

👤 Прізвище Ім'я
   ДН: дд.мм.рррр | OneBox id: XXXXX
   Перший візит: дд.мм.рррр | Останній: дд.мм.рррр

💰 Статистика покупок:
   Чеків: N | Сума: XX XXX грн
   Середній чек: X XXX грн
   За 30 днів: N чеків / X XXX грн   ← тільки якщо є

🏪 Магазини: River Mall (6), Лесі Українки (4)

🧾 Останні покупки:
   • дд.мм.рр — Магазин — X XXX грн
   ...

📱 OneBox: Прізвище Ім'я, id=XXX, bdate=дд.мм.рррр
```

Також доступна з inline-меню "⚙️ Технічний стан → 👤 Покупець".

### ⚙️ Технічний стан
Показує inline-меню з 5 кнопок:
- **📊 Статус черги** — лічильники по statuses + час останнього sync
- **❌ Failed чеки** — останні 10 чеків з помилкою
- **📅 Дайджест** — зведення за вчора
- **🔄 Синхронізація ДН** — статус backfill дат народження
- **👤 Покупець** — перехід до пошуку клієнта

### /fix_bdate 380XXXXXXXXX
Передає дату народження клієнта з BAF в OneBox вручну.

### /users
Список усіх авторизованих юзерів бота з їх ролями.

### /backfill_status
Статистика синхронізації дат народження: скільки клієнтів з ДН, скільки синхронізовано в OneBox.

## Технічна архітектура бота

```
Telegram ──── aiogram 3 (long-polling)
                │
                ├── bot.py         FSM, хендлери, клавіатури
                ├── roles.py       RBAC, авторизація
                ├── handlers.py    sync DB-запити (run_in_executor)
                └── cash.py        1С API запит

config/roles.yml  ←── live-reload при кожній авторизації
data/bot_users.json  ←── persistent store авторизованих юзерів
```

### FSM (Finite State Machine)

Стан `CustomerLookup.waiting_phone` використовується для збору номера телефону:
- активується при натисканні кнопки "👤 Покупець" або команді `/customer` без аргументу
- наступне текстове повідомлення від юзера трактується як номер

### Нормалізація телефону

Формат: `+380XXXXXXXXX` (E.164).
Підтримуються входи: `380...`, `0...`, `+380...`.

## Troubleshooting

| Проблема | Причина | Рішення |
|---|---|---|
| Бот не відповідає на авторизацію | Невалідний `config/roles.yml` | Перевірити YAML синтаксис (`owner:` з новим рядком і `- "+380..."`) |
| "номера немає у списку" | Телефон не доданий в roles.yml | Додати номер, зберегти — діє одразу |
| Меню пропало після відповіді | `ForceReply` скидає клавіатуру | Фіксовано: `_run_sync` завжди повертає `reply_markup=_kb_for(role)` |
| Кнопка "👤 Покупець" → помилка | Текст кнопки парсився як номер | Фіксовано: args беруться тільки для `/customer` команди |
| "Технічний стан мовчить" | Відсутній дозвіл `/status` в ролі | Перевірити `ROLE_COMMANDS` в `roles.py` |
