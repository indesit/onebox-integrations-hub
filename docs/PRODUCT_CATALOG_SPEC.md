# Специфікація каталогу товарів

## Канонічна модель
- **Ключ варіанту товару:** `(product_uuid, characteristic_uuid)` -> `product_catalog` -> `article`
- **Проблема розмірів:** BAF зберігає "базовий" артикул (напр. `429-078-QAS`) для товару, а розмір — у `characteristic_name` (напр. "рожевий, XS").
- **Рішення для OneBox:** OneBox вимагає унікального артикулу для кожної варіації (напр. `429-078-QAS-XS`). Тому при підготовці payload для OneBox, ETL-шар або воркер повинен виконувати екстракцію суфікса з `characteristic_name` і конкатенувати його з `article`.

## Поля (BAF)
- `product_uuid`: GUID товару
- `product_name`: Назва
- `article`: Базовий артикул (без розміру)
- `characteristic_uuid`: GUID характеристики (може бути пустим для безрозмірних товарів)
- `characteristic_name`: Текстовий опис (містить колір і розмір)
